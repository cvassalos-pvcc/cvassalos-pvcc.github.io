  1. Explain why natural language is not an effective tool for describing algorithms.
    Natural language is too nuanced, and involves too much subjectivity for a computer to understand. A word can have multiple meanings and be used in many different ways, and a computer is not capable of this kind of logic.   2. Problem 1:
    A student has three grades; the final grade is the average of the the three. List the algorithmic steps to find the final grade.
    Step 1: Add the three grades together.
    Step 2: Divide the sum of the grades by three.
    Resulting calculation gives the average of the three grades.
  3. Problem 2:
    A car owner recorded the car's previous car mileage, current mileage, and the number of gallons of gas the owner filled their tank with. List the algorithmic steps to find the car's mpg (miles per gallon).
    Step 1: Subtract the current mileage from the previous car mileage.
    Step 2: Take resulting value and divide by the number of gallons of gas the owner filled the tank with.
    Resulting calculations gives you the MPG.

Chapter 2: Briefly define:

  1. natural language - the language we speak and read in everyday life
  2. pseudocode - an informal way of writing code for an algorithm. Meant for the coder to understand and not the computer.
  3. sequential operation (straight-line algorithm) - an algorithm that executes from top to bottom, straight through
  4. input - providing code with data values from the outside world to be used in the computations
  5. output - the computer sending the resulting computations to the outside world
  6. conditional statement - operations that ask a true or false questionto determine which path through the algorithm to take
  7. boolean condition - an expression that evaluates to either "true" or "false"
  8. algorithmic discovery process - finding the solution to a given problem

Chapter 3: Briefly describe of define:

  1. why algorithms should be correct - an algorithm, no matter how elegant or efficient, if wrong, is completely useless
  2. why algorithms should be efficient - while modern computers have a lot more computing power, it is still good practice to carefully use the computing resources at hand
  3. program maintenance - a program must be maintained after repeated use, to fix errors uncovered through user, or to expand the program's functionality.
  4. importance of algorithm: ease of understanding - if an algorithm is not well commented, or well written, any changes or fixes that may need to be made are made substantially harder and take longer because the programmer has to figure out how everything works first. "Spaghetti code" is a term used for algorithms that are very hard to understand.
  5. algorithmic elegance - the algorithmic equivalent of style
  6. sequential search algorithm - an algorithm that searched through every item in a list, until it finds the desired outcome
  7. order of magnitude - this is how a computer scientist measures the efficiency of a program
  8. selection sort - this is a type of algorithm that sorts through a list of items
  9. flop (floating point operation) - a unit measuring how many arithmetic operations a computer can do in one second
  10. teraflop - 16 trillion floating-point operations per second
  11. petraflop - 93 quadrillion floating-point operations per second
  12. parallel processor - computing where many computations are taking place at the same time.
  13. data cleanup problem - this is an issue where you need to potentially remove data declared in the code before making calculations
  14. binary search algorithm - a search algorithm that is more efficient, but only works if the list is already sorted. It starts in the middle of the list and searches up or down based on if the value is greater than or less than the value being searched.
  15. pattern matching - an algortihm to searches for a specific string in text

Carl Friedrich Gauss

Carl Friederich Gauss was a brilliant German mathematician and physicist who made many great contributions to many scientific fields, he was sometimes referred to as "Princeps mathematicorum", which is Latin for "the foremost of mathematicians". There is a famour story about Gauss, involving him as a student in school. The instructor asked all the students to figure out the sum of number 1 through 100 in an attempt to keep the students quiet for a while. Gauss ruined this plan by responded quickly that the sum is 5,050. He figured it out not by doing the arithmetic, but by figuring out that if you "fold" the numbers (1 + 100, 99 + 2, etc) they would each add up to 101, and 50 pairs of 101 is 5,050. A brilliant way for a child, or really any one at the time, to solve this problem. Gaussian optics is one of Gauss' many contributions to the scientific world. It is a framework to describe optical phenomena. The Gaussian blur, named after Carl Fredrick Gauss, uses a mathematical function to blur a picture
