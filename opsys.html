Operating Systems Terminology
TERM Definition
Operating System the program that runs and manages the computer and all its components, it also provides a graphical user interface for humans to use.
Job Queue a data structure that contains the order of the jobs (processes) that the operating system needs to run to function
FIFO First In, First Out; it is a method of handling elements of an operating system.
interactive processing This is when the computer processes data as it is being used
time-sharing sharing the resources of a computer amongst multiple users
multiprogramming when more than one program runs at a time
application software Programs on your computer designed to do specific tasks for the user. Examples are: Visual Studio Code, Adobe Photoshop, Mozilla Firefox
system software a collection of computer programs that manage the resources of a computer and facilitates access to those resources
utility software collection of libraries providing services to the user and the system routines. Some examples are: text editors, online help routines, image and sound applications, and control panels
Graphical User Interface Also known as the GUI, provides an easy to understand visual overview of the computers functions and capabilities
shell A UNIX term for the interactive user interface with an operating system
kernel A program that is on all computers that has complete control over the whole system and does things such as running processes and managing hardware devices
device driver Manages hardware components and any peripherals that may be attached to the computer
file manager A computer program that provides a user interface to manage files and folders
memory manager The process of controlling and coordination a computer's main memory
virtual memory Uses both hardware and software to enable a computer to compensate for physical memory shortages
boot strapping To boot or to load a program into a comptuer using a much smaller inital program to load in the desired program, which is usually an OS
ROM a specific type of RAM where the data stored in the memory is recorded during production and can not be modified or removed, often used to hold important system instructions
deadlock A condition that may happen in a system composed of multiple processes that can access shared resources
spooling The practice of holding data in temporary storage for execution by another device or program
super user (or administrator) A user that is authorized to perform security-relevant functions that most users are not allowed to perform
multi-factor authentication Having two forms of security (more than just a password), to access and account
