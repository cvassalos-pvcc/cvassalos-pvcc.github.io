Chapter 9: Introduction to High Level Languages

List four disadvantages of using assembly language:

  1. The programmer must "manually" manage the movement of data items bewteen and among memory locations and registers (although such data items can be assigned mnemonic names).
  2. The programmer must take a microscopic view of a task, breaking it down into tiny subtasks-ADD, COMPARE, INCREMENT-that direct what is going on in individual memory locations and registers.
  3. An assembly language program is machine specific.
  4. Statements are not natural-language-like (although operations are given mnemonic code words as an improvement over a string of bits).

List four expectations programmers have for writing programs in a high-level language ("third generation" language):

  1. The programmer need not manage the details of the movement of data items within memory or pay any attention to exactly where those items are stored.
  2. The programmer can take a macroscopic view of tasks, thinking at a higher level of problem solving (add B and C, and call the result A). The "primitive operations" used as building blocks in algorithm construction can be larger.
  3. Programs are portable rather than machine specific.
  4. Programming statements are closer to natural language and use standard mathematical notation.

Briefly describe or define:

  1. object code - assembly language instructions translated into binary machine language instructions
  2. source code - high level language instructions
  3. procedural language - a type of computer programming language that specifies a series of well-structured steps and procedures within its programming context to compose a program
  4. Integrated Development Environment (IDE) - a software application that helps a user to write code
  5. software development life cycle - the multiple steps it takes to create a fully functional piece of software
  6. waterfall model of software development - a project being broken into numerous parts and completing them sequentially
  7. agile model of software development - problem specifications are never a "done-deal", so developers must be quick and ready to fix and/or change any code in the project

Describe each of these components of the software development life cycle:

  1. feasibility study - evaluates a proposed project and compares the costs and benefits of various solutions
  2. problem specification - developing a clear, concise, and unambiguous statement of the exact problem the software is to solve
  3. program design - planning how the program is going to the solve the exact problem
  4. algorithm selection - once the various subtasks have been identified, algorithms must be found to carry them out
  5. coding - the process of c onverting the detailed designs into computer code
  6. debugging - the process of locating and correcting program errors
  7. testing - testing and benchmarking the code throughout the softwares development to make sure there are no errors and everything is working as intended
  8. verification - can be used to prove that if the input data to a program satisfies certain conditions, then, after the program has been run on this data, the output data satisfies certain other conditions
  9. documentation

Questions & Answers:

  • Why does the textbook refer to "the Tower of Babel" regarding computer languages?
  • Similar to the when the Tower of Babel fell, instead of there being just one computer language, there are numerous computer languages
  • Why isn't there just one computer programming language?
  • The way computer programming languages are designed, some are better for a particular thing, such as accomplishing a certain type of program, or using the smallest amount of memory
  • Why didn't FORTRAN use the less than ( < ) symbol when comparing two numbers?
  • Because the way data was fed to the FORTRAN machines was via punch cards, and in that time there was no "<" for punch cards
  • What was COBOL designed for?
  • "Common Business-Oriented Language" was designed to be a language for managers to use in business for such things as managing inventories and payrolls
  • What was the language C designed for?
  • It was orginally designed to write the operating language UNIX
  • What is the difference between C and C++?
  • C++ is the same as C but with many new features, the most significant one being the ability to do object-oriented programming
  • Who or what is the language Ada named for, and why?
  • The language Ada was named after Ada Augusta Byron Lovelace, the daughter of the poet Lord Byron and later the wife of Lord Lovelace
  • What is a Java "applet"?
  • Java applets are small programs designed to run from webpages

  • R is a specialized programming language designed for statistical computing and graphics
