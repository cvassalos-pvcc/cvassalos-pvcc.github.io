Von Neumann computer architecture, 4 major subsystems

  1. memory
  2. input/output
  3. arithmetic/logic unit (ALU)
  4. control unit

Von Neumann computer architecture, 3 characteristcs:

  1. There are four subsystems that work together.
  2. Instructions on what to be done are stored in memory in binary, also known as the "stored program" concept.
  3. One instruction at a time is sent to the CPU to be decoded and executed, also known as the "sequential execution of instructions".

Computer Architecture Terms

  1. RAM - memory is stored here cells, each with a unique address, in binary to be decoded and executed
  2. cache - smaller, but faster than RAM, this is were data based on the "Principle of locality" is sent to from the RAM before being processed in order to get maximum speed out of the computer
  3. Principle of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. It will access that same instruction or piece of data in the very near future.
    2. It will soon access the instructions or data that are located near that piece of data, where "near" means an address whose numerical value is close to this one.
  4. ROM - a specific type of RAM where the data stored in the memory is recorded during production and can not be modified or removed, often used to hold important system instructions`
  5. memory cell size - RAM is made up of multiple memory cell, and each cell has a specfic size measured in bits
  6. memory cell address - this is the unique identifier of a cell in the RAM where data is stored
  7. maximum memory size - the maximum number of bits that can be stored in a memory cell. Commonly was 32 bit, but in modern computers, 64 bit is the standard
  8. register - a storage cell that is specified to hold the operands of arithmetic operations, as well as the results after processing
  9. fetch - an operation that gets data from a memory cell and puts it into the memory data register (MDR)
  10. store - an operation that puts the data from the MDR into a memory cell
  11. memory access time - the time to carry out a fetch or store operation
  12. DASD - an acronym for "direct access storage device", it is a storage device where every memory cell has a unique address, but the time to access the data varies based on multiple variables, such as physical location or current state of the device.
  13. Arithmetic and Logic Unit (ALU) - a major subset of the Von Neumann computer architecture, this unit handles mathematic and logical operations. In most systems it has been integrated with the control unit into the CPU of a computer
  14. Control Unit - has control of other units to perform three tasks repeatedly: fetch, decode, execute

Chapter 6:

Four things the interface between the computer and hardware does:

  1. Hides from the user the messy details of the underlying hardware
  2. Presents information about what is happening in a way that does not rquire in-depth knowledge of the internal structure of the system
  3. Allows easy user access to the resources available on this computer
  4. Prevents accidental or intentional damage to hardware, programs, and data

Responsibilities of systems software:

  1. Hides the complex and unimportant (to the user) details of the internal structure of the Von Neumann architecture
  2. Presents important information to the user in a way that is easy to understand
  3. Allows the user to access machine resources in a simple and efficient way
  4. Provides a secure and safe environment in which to operate

Terms:

  1. systems software - a collection of computer programs that manage the resources of a computer and facilitates access to those resources
  2. user interface - the graphical user interface (GUI) provides an easy to understand visual overview of the computers functions and capabilities
  3. language services - programs that allow the user to write programs in high-level language (like english) instead of machine code. Some of these programs are assemblers, compilers, and interpreters
  4. memory managers - these are programs that allocate memory for programs
  5. information managers - programs that handle organization, storage, and retrieval of information
  6. i/o managers - software packages that make input and output devices compatible with the computer
  7. scheduler - a system program that keeps a list of programs for the processor to run, enabling the computer to run multiple programs at once
  8. utilities - collection of libraries providing services to the user and the system routines. Some examples are: text editors, online help routines, image and sound applications, and control panels
  9. assembly language - one of the first non-machine languages created, it created a more productive and user-oriented environment

Functions of an Operating System

  1. User interface management
  2. Control of access to the system and to data files
  3. Program scheduling and activation
  4. Efficient resource allocation
  5. Deadlock detection and error detection
